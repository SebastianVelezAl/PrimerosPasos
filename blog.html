Comandos Conoce el uso de consola de Git.

¡Domina los comandos y lleva tu desarrollo al siguiente nivel!

Conocer los comandos de Git es fundamental para cualquier desarrollador o equipo de desarrollo que desee utilizar este sistema de control de versiones de manera efectiva. Estos comandos son las herramientas clave que nos permiten realizar operaciones fundamentales en nuestros proyectos y mantener un flujo de trabajo ordenado y colaborativo.

conocer y dominar los comandos de Git nos brinda un mayor control sobre nuestro código, nos permite colaborar de manera efectiva y nos facilita el manejo de versiones en nuestros proyectos de desarrollo. Es una habilidad esencial para cualquier desarrollador que desee trabajar de manera eficiente y profesional.

Comando Complemnto Acción
int Inicializar el repositorio.
add nombre_de_archivo.extensión. agregar el archivo al repositorio.
commit -m "mensaje" Agregar los cambios en la rama que nos encontramos.
status Visualizar los documtnos que tienen cambios.
log nombre_de_archivos.extensión histórico de cambios con detalles
pull Traer repositorio remoto.
push envía a otro repositorio remoto lo que estamos haciendo.
ls listado de carpetas en donde me encuentro.
pwd ubicación actual.
mkdir make directory nueva carpeta.
touch archivo.extensión. crear archivo vacío.
cat archivo.extensión. muestra el contenido del archivo.
history historial de comandos utilizados durante esa sesión.
rm archivo.extensión. Eliminación de archivo.
--help ayuda sobre el comando.
checkout nombre de la rama traer cambios realizados a una rama en asiganda.
config --list muestra la lista de configuración de git.
diff Saber que se modifico en un archivo entre dos versiones
pendiente pendiente definir información.
